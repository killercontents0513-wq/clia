# Amazon A+ Content Builder
### LG Electronics — Retail OBS Contents Automation

> 아마존 A+ 콘텐츠 페이지를 누구나 쉽게 만들 수 있는 웹 기반 빌더 도구입니다.
> 이미지 또는 동영상 URL만 있으면 Figma 템플릿에 맞게 자동으로 리사이징하고, 모듈을 조합해 완성된 A+ 페이지를 제작할 수 있습니다.

---

## 이 도구가 해결하는 문제

아마존 A+ 콘텐츠는 제품 상세 페이지를 풍부하게 꾸밀 수 있는 기능입니다.  
하지만 각 모듈마다 **요구하는 이미지 크기가 다르고** (예: 970×400, 500×460, 300×300 등),  
어떤 모듈이 있는지조차 모르는 상태에서 콘텐츠를 만드는 것은 매우 번거롭습니다.

이 도구는 다음을 자동화합니다:
- 이미지/동영상 소스 → 각 템플릿에 맞는 크기로 자동 적용
- 14가지 공식 A+ 모듈 템플릿을 시각적으로 미리보기
- 원하는 모듈을 조합해 완성 페이지를 실시간으로 확인
- 완성된 페이지를 HTML 파일로 저장

---

## 사용 방법 (User Flow)

```
① 이미지 URL 또는 동영상 URL 입력
        ↓
② (동영상인 경우) 원하는 장면에서 프레임 캡처
        ↓
③ 우측 갤러리에서 원하는 모듈 선택
   - 싱글 이미지 모듈 → 바로 "페이지에 추가"
   - 멀티 이미지 모듈 → 슬롯별 이미지 설정 후 추가
        ↓
④ 좌측 미리보기에서 완성된 A+ 페이지 실시간 확인
   (↑↓ 버튼으로 순서 변경, ✕ 버튼으로 삭제)
        ↓
⑤ "HTML 저장" 버튼으로 최종 파일 다운로드
```

## 기술 스택 (Tech Stack)

> 💡 **초보자를 위한 설명**: 기술 스택이란 이 도구를 만드는 데 사용된 재료 목록입니다.

### 사용한 기술

| 기술 | 역할 | 쉬운 설명 |
|------|------|-----------|
| **HTML** | 화면 구조 | 건물의 뼈대 (벽, 문, 창문 위치) |
| **CSS** | 화면 디자인 | 인테리어 (색상, 크기, 배치) |
| **JavaScript** | 동작 로직 | 전기 시스템 (버튼 누르면 무슨 일이 일어나는지) |
| **HTML5 Video API** | 동영상 재생 | 브라우저 내장 동영상 플레이어 |
| **Canvas API** | 이미지 처리 | 보이지 않는 가상의 도화지 (동영상 프레임 추출에 사용) |
| **Figma MCP** | 템플릿 데이터 | Figma 디자인 파일에서 템플릿 규격을 자동으로 읽어옴 |
| **CSS Transform** | 미리보기 축소 | 970px짜리 실제 크기 모듈을 화면에 맞게 자동으로 줄여서 표시 |

### 특별한 점: 외부 라이브러리 없음 (Zero Dependency)

이 도구는 React, Vue 같은 프레임워크나 npm 패키지를 **전혀 사용하지 않습니다.**  
HTML 파일 하나를 브라우저에서 열면 그대로 작동합니다.  
인터넷 연결 없이도 동작하며, 설치가 필요 없습니다.

## 시스템 작동 원리

### 1. 이미지 로딩

```
사용자가 URL 입력
    → 브라우저가 해당 주소의 이미지를 불러옴
    → 성공 시: 모든 템플릿 카드에 이미지 자동 적용
    → 실패 시: CORS 정책으로 인해 미리보기는 불가하지만 URL은 저장됨
```

> 💡 **CORS란?** 다른 사이트의 파일을 가져올 때 보안상 제한이 걸리는 브라우저 정책입니다.  
> 직접 소유한 서버의 이미지나 CORS를 허용한 서비스의 이미지는 완전하게 미리보기가 됩니다.

### 2. 동영상 프레임 추출

```
동영상 URL 입력 (.mp4, .webm 등 확장자 자동 감지)
    → 브라우저 내장 비디오 플레이어로 로드
    → 타임라인 스크러버로 원하는 장면으로 이동
    → "프레임 캡처" 버튼 클릭
    → Canvas(가상 도화지)에 해당 프레임을 그림
    → JPG 이미지 데이터로 변환 → 이미지처럼 사용
```

### 3. 템플릿 렌더링

```
Figma 파일에서 읽어온 14개 모듈의 치수 정보
    → JavaScript가 각 모듈의 HTML 코드를 생성
    → 실제 크기(970px 기준)로 만들어진 뒤
    → CSS transform: scale()로 화면에 맞게 자동 축소 표시
    → 실제 비율을 유지하면서 정확한 미리보기 제공
```

### 4. 멀티 슬롯 모달

```
멀티 이미지 모듈 선택 시
    → 슬롯 수만큼 URL 입력창이 있는 팝업 창 등장
    → 각 슬롯에 이미지 또는 동영상 URL 입력 가능
    → 동영상 URL 입력 시: 슬롯 내부에 미니 비디오 플레이어 표시
    → 프레임 캡처 후 슬롯에 적용
    → "페이지에 추가" 클릭 시 슬롯별 이미지로 모듈 완성
```

### 5. 페이지 빌더

```
페이지 = JavaScript 배열 [ 모듈1, 모듈2, 모듈3, ... ]
    각 모듈은 { 타입, 이미지URL, 슬롯이미지맵 } 형태로 저장
    → 배열이 바뀔 때마다 화면을 새로 그림 (실시간 미리보기)
    → 순서 변경 = 배열에서 항목 위치를 바꾸고 다시 그림
    → 삭제 = 배열에서 해당 항목 제거 후 다시 그림
```

### 6. HTML 내보내기

```
"HTML 저장" 클릭
    → 모든 모듈의 HTML 코드를 하나로 합침
    → CSS 스타일도 함께 포함된 완전한 HTML 파일 생성
    → 브라우저 다운로드 기능으로 파일 저장
    → 이 파일 하나로 아마존에 업로드하거나 공유 가능
```

## 모듈 구성 (Figma 템플릿 기반)

Figma 파일: **Retail OBS Contents Automation**  
총 **14개** 모듈 / 3가지 카테고리

### 싱글 이미지 (이미지 1개)
| 모듈명 | 이미지 크기 |
|--------|-------------|
| Standard Single Image & Sidebar | 500×460 |
| Standard Single Left Image | 300×300 |
| Standard Single Right Image | 300×300 |
| Standard Single Image & Highlights | 500×560 |
| Standard Image & Light Text | 970×400 |
| Standard Image Dark Text Overlay | 970×300 |

### 멀티 이미지 (슬롯별 설정)
| 모듈명 | 슬롯 구성 |
|--------|-----------|
| Standard Three Image & Text | 310×300 × 3개 |
| Standard Four Image & Text | 180×180 × 4개 |
| Standard Four Image Highlights | 메인 400×560 + 서브 140×230 × 3개 |
| Standard Four Images Text Quadrant | 160×200 × 4개 (2×2 그리드) |
| Standard Comparison Table | 260×200 × 3개 (제품 비교) |

| 모듈명 | 내용 |
|--------|------|
| Standard Text | 헤드라인 + 본문 텍스트 |
| Standard Single Image & Specs Detail | 이미지 + 사양 목록 |
| Standard Technical Specifications | 기술 사양 테이블 |

```

├── amazon-aplus/
│   └── index.html       ← 이 도구의 전체 코드 (HTML + CSS + JS 통합)
```

## 제약 사항 및 유의점

| 상황 | 내용 |
|------|------|
| **외부 이미지 CORS** | 이미지를 소유한 서버가 허용해야 캔버스 추출 가능. 미리보기는 대부분 가능 |
